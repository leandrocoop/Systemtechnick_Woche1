Azure-Struktur: Reihenfolge & Erklärung

1. Management Groups (MG)

Was ist das?
Oberste logische Ebene zur Strukturierung mehrerer Subscriptions.

Wofür?
Ideal für grössere Organisationen. Ermöglicht z. B. zentrale Richtlinien oder IAM-Rollen.

Beispiel:
Management Group test, darunter test-sandboxes

2. Subscriptions

Was ist das?
Eine Subscription ist eine Abrechnungseinheit in Azure. Alle Ressourcen kostenpflichtig laufen innerhalb einer Subscription.

Wofür?
Trennung von z. B. Produktion, Entwicklung oder Kundenumgebungen.

Beispiel:
test-001 ist eine Subscription unter test-sandboxes

3. Resource Groups (RG)

Was ist das?
Eine logische Gruppierung von Azure-Ressourcen.

Wofür?
Ermöglicht gemeinsame Verwaltung, z. B. Backup, Tags, Berechtigungen.

Beispiel:
rg-webapp-prod enthält Web App, Application Insights
rg-storage-test enthält Storage Account, Key Vault

4. IAM / RBAC

Was ist das?
Zugriffskontrolle: Wer darf was tun und wo?

Wofür?
RBAC erlaubt gezielte Rollenzuweisung (z. B. Reader, Contributor).

Wo anwenden?
Auf Management Group, Subscription, RG oder einzelne Ressourcenebene.

Beispiel:
Max Mustermann – Reader auf test-001
Gruppe Dev-Team – Contributor auf rg-webapp-prod

🌐 Azure-Zugriffsstruktur (Beispiel Jason)

Management Group Struktur:
Jason ist Teil des Tenants, aber hat keinen Zugriff auf die Tenant Root Group oder übergeordnete Management Groups (test, test-sandboxes).
Zugriff beginnt auf Subscription-Ebene (test-001).

Subscription: test-001
Jason hat Reader-Rolle auf der gesamten Subscription.
Innerhalb dieser Subscription sind mehrere Resource Groups enthalten.

Resource Groups:
Sichtbare RGs:
rg-webapp-prod: Enthält Web App + Monitoring.
rg-storage-test: Enthält Storage Account + Key Vault.
Jason kann Inhalte lesen, aber nicht verändern.

IAM / RBAC:
Rollenvergabe auf Subscription-Ebene:
Jason: Reader
Dev-Team: Contributor
Keine individuellen Rollen auf RG-Ebene sichtbar (vermutlich zentral vergeben auf Subscription).

Speicherkonto & VM in Azure erstellen

Grundlagen

Speicherkonto:
Ein Azure-Dienst zum Speichern von Daten wie Dateien, Backups oder Blobs.

VM (Virtuelle Maschine):
Ein virtueller Computer in der Cloud mit wählbarem Betriebssystem, CPU, RAM.

Beides sind Ressourcen in Azure, die innerhalb einer Ressourcengruppe angelegt werden. Eine VM kann Daten in einem Speicherkonto ablegen – z. B. für Backups oder Logdateien.

1. Speicherkonto erstellen

  • Im Abonnement auf „Ressourcen“ klicken.
  • „Speicherkonto“ erstellen und eine bestehende Ressourcengruppe auswählen.
  • Region: West Europe
  • Leistung: Standard
  • Replikation: LRS (Local Redundant Storage) – günstig & einfach.
  • Auf „Überprüfen + Erstellen“ klicken → dann „Erstellen“.

2. Virtuelle Maschine (VM) erstellen

  • Im Abonnement auf „Ressourcen“ klicken.
  • „Virtueller Computer“ erstellen und eine bestehende Ressourcengruppe auswählen.
  • Region: West Europe
  • Image: Ubuntu 22.04 LTS – kostenlos & leichtgewichtig
  • Grösse: Standard D2s v6
  • Authentifizierung: SSH oder Passwort
  • Öffentliche Ports: Port 22 für SSH öffnen
  • OS-Datenträgertyp: Standard HDD – günstige Option
  • Auf „Überprüfen + Erstellen“ klicken → dann „Erstellen“.

Zugriff per PuTTY auf die VM

Um dich per SSH mit deiner Azure-VM zu verbinden, kannst du das Tool PuTTY verwenden. Dafür brauchst du die öffentliche IP-Adresse deiner VM und dein Key-Pair (.ppk).

PuTTY einrichten

  • Host Name: Öffentliche IP der VM
  • Port: 22
  • Connection Type: SSH

Authentifizierung konfigurieren

Gehe im linken Menü zu:

  • Connection > SSH > Auth > Credentials

Klicke dort auf „Browse…“ und wähle deine .ppk-Datei (Key-Pair) aus.

Anschliessend kannst du auf „Open“ klicken und dich mit der VM verbinden.

Azure Netzwerkgrundlagen – Anleitung & Dokumentation

1. Virtuelles Netzwerk (VNet)

Was ist ein VNet?
Ein virtuelles Netzwerk (VNet) ist ein privates, logisches Netzwerk in Azure – vergleichbar mit einem lokalen Netzwerk im Rechenzentrum.

Es bildet die Grundlage für die Kommunikation zwischen Azure-Ressourcen wie virtuellen Maschinen.

2. Netzwerksicherheitsgruppen (NSGs)

Was ist eine NSG?
Eine Netzwerksicherheitsgruppe (NSG) ist eine Sammlung von Regeln, die den ein- und ausgehenden Netzwerkverkehr für Azure-Ressourcen steuern.

Sie erlaubt z. B. gezielten Zugriff auf Ports oder das Blockieren bestimmter IP-Adressen.

3. VNet-Peering

Was ist VNet-Peering?
VNet-Peering verbindet zwei virtuelle Netzwerke, sodass Ressourcen sicher miteinander kommunizieren – ohne über das öffentliche Internet zu gehen.

🔧 Schritt-für-Schritt-Anleitung

1.1 Virtuelles Netzwerk erstellen

  • Azure-Portal → „Virtuelle Netzwerke“ → „+ Erstellen“
  • Ressourcengruppe: auswählen
  • Name: VNet-DEMO
  • Region: West Europe
  • IP-Adressbereich: 10.0.0.0/16
  • Subnetz hinzufügen:
    • Name: Subnet-DEMO
    • Adressbereich: 10.0.1.0/24
    • Network security group: auswählen
  • → „Überprüfen + Erstellen“ → „Erstellen“

1.2 NSG erstellen und konfigurieren

  • Azure-Portal → „Netzwerksicherheitsgruppen“ → „+ Erstellen“
  • Name: NSG-DEMO
  • Ressourcengruppe & Region auswählen
  • → „Erstellen“

1.3 NSG mit Subnetz verknüpfen

  • Öffne NSG-DEMO
  • Reiter „Subnetze“ → „+ Subnetz zuordnen“
  • VNet: VNet-DEMO, Subnetz: Subnet-DEMO
  • → „OK“

1.4 Zweites VNet für Peering erstellen

  • Wie Schritt 1.1 – aber:
  • Name: VNet-DEMO-2
  • IP-Adressbereich: 10.1.0.0/16
  • Subnetz: 10.1.1.0/24
  • Achte darauf: Adressbereiche dürfen sich nicht überschneiden!

1.5 VNet-Peering einrichten

  1. In VNet-DEMO:
    Öffne VNet-DEMO → Reiter „Peering“ → „+ Hinzufügen“
    Name: PeeringToVNet2, Ziel-VNet: VNet-DEMO-2
    Kommunikation zulassen
  2. In VNet-DEMO-2:
    Wiederhole die Schritte:
    Name: PeeringToVNet1, Ziel-VNet: VNet-DEMO
    Kommunikation zulassen

App Services & Functions in Azure

Was ist ein App Service?

Ein App Service in Microsoft Azure ist ein Platform-as-a-Service (PaaS)-Angebot, das dir ermöglicht, Webanwendungen, APIs oder mobile Backends schnell und ohne grossen Infrastrukturaufwand bereitzustellen und zu betreiben.

Vorgehen zur Erstellung

  • Im Azure-Portal auf „Subscriptions“ klicken und passende auswählen.
  • Dann auf „Resource Groups“ gehen und eine Gruppe auswählen.
  • Klicke auf „+ Create resources“.
  • Suche nach „App Services“ und wähle eine Option aus.
  • Gehe den Setup-Prozess durch (Name, Region, Plan etc.).
  • Erstelle den Service und teste die Bereitstellung.
  • Deploye deine App via z. B. GitHub oder lokalem Code-Upload.

Deployment einer statischen HTML/CSS-Webseite mit Azure Static Web Apps

Ziel

Bereitstellung einer statischen Webseite (HTML, CSS) über Azure Static Web Apps mit automatischem Deployment via GitHub.

1. Voraussetzungen

  • Ein GitHub-Repository mit HTML/CSS-Dateien (z. B. index.html, style.css)
  • Ein Azure-Konto
  • Zugriff auf Azure Static Web App erstellen

2. Einrichtung in Azure

  • Azure-Portal → "Statische Web-App" erstellen
  • Abonnement: Test
  • Ressourcengruppe: TEest
  • Name: Testguide
  • Hostingregion: Global
  • Plan: Standard (nicht kostenlos)
  • Quelle: GitHub, dann Repository und Branch auswählen

3. Automatisches Deployment via GitHub

Azure erzeugt beim Erstellen eine Datei im Repository:

.github/workflows/azure-static-web-apps-XYZ.yml

Sie beschreibt den GitHub Actions Workflow für das Deployment. Änderungen am Code → neue Version wird automatisch deployed.

4. Ergebnis

  • Statische Webseite ist über eine öffentliche Azure-URL erreichbar
  • Ideal für reine HTML/CSS-Seiten

5. Kosten

  • Standard-Plan: Kostenpflichtig – für produktive Umgebungen
  • Free-Plan: Für Hobby- oder Testprojekte ausreichend

Geräteübersicht Logistik & Verkauf

1. MDE-Gerät

Ein mobiles Datenerfassungsgerät (z. B. von Zebra oder Datalogic). Es wird in Verkaufsstellen für Aufgaben wie Wareneingang, Lagerverwaltung, Preisprüfung, Inventur und Etikettendruck verwendet. Es ist robust gebaut und besitzt Touchscreen & Tastatur für flexible Eingaben.

2. Passabene-Scanner

Ein Handscanner für Self-Scanning durch Kundschaft. Kunden scannen Produkte selbst während des Einkaufs. Das Gerät kommuniziert mit der Kasse, um am Ende direkt zu bezahlen. Im Bild ist ein Zebra-Gerät mit dem Status „Ausser Betrieb“ zu sehen.

3. Liquidationsdrucker

Ein mobiler Etikettendrucker (z. B. Zebra ZQ610). Wird eingesetzt, um Preisetiketten vor Ort auszudrucken – vor allem für Aktionen, Preisreduktionen oder Lagerkennzeichnungen. Verbindung per Bluetooth oder WLAN.

4. Logistikergerät

Ein industrietaugliches Gerät mit physischer Tastatur und ergonomischem Scan-Griff – speziell für Lagerlogistik. Es wird zur Kommissionierung, beim Warenein- und -ausgang sowie für Bestandskontrollen eingesetzt. Dank robuster Bauweise und langer Akkulaufzeit ideal für den Dauereinsatz in der Intralogistik.

5. Logistik-Fingerscanner

Ein kompakter, am Finger tragbarer Barcode-Scanner. Ideal für Reichweitenlogistiker, die beide Hände frei benötigen. Wird mit einem Hostgerät (z. B. Smartwatch oder MDE) gekoppelt und beschleunigt Kommissionierung und Lagerprozesse erheblich.

6. Gabelstapler-Terminal

Ein fest verbautes Industrie-Tablet (z. B. Jungheinrich WMT 110) im Gabelstapler. Es dient zur Navigation, Auftragsanzeige und Lagersteuerung. Die Geräte sind robust und oft mit WLAN, Scanner, USB und Touch ausgestattet.
